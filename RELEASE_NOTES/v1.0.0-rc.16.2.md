### Signal Persistence on API Errors

Fixed an issue where the volume signal animation would reset to neutral (no animation) when the history API temporarily failed or returned insufficient data.

- Every 60s refresh would call `updateVolumeSignal`
- If history API failed or returned no data â†’ signal = "neutral"
- All blink classes were removed, animation stopped
- Default hover styles (green) would then show

#### The Fix
- Added `_lastVolumeSignal` state variable to track last valid signal
- If current calculation returns "neutral" but we had a previous valid signal â†’ keep it
- Animation only changes when we have NEW valid data
- Truly neutral market conditions still show no animation
