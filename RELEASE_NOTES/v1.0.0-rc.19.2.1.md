# v1.0.0-rc.19.2.1 (2025-12-04)

Small UX polish + tooltip consistency fixes applied after RC19.2.

### Fixed
- Ensure API Status tooltip chips render in the requested order: **Bittensor SDK**, **Taostats**, **CoinGecko** on initial load and during live updates.
- Preserve richer HTML tooltip content unless explicitly missing; avoid JS overwriting long HTML descriptions.
- Reduce visual prominence of the main API status chip (smaller padding, reduced font size) to improve card hierarchy.

### Changed
- The API Status card now displays a single, centered status chip (OK / Partial / Error). Detailed per-source chips are visible in the info-badge tooltip.
- Frontend tooltip engine extended to support trusted HTML tooltips via `data-tooltip-html="true"`.

### Notes
- Bumped `VERSION` to `v1.0.0-rc.19.2.1` and pushed annotated tag `v1.0.0-rc.19.2.1`.
- Please hard-refresh the dashboard or purge CDN cache to see the latest assets if you still see the old tooltip ordering.
