### Persistent Blink Animation Fix

Fixed an issue where the volume signal border animation would revert to green (default hover color) instead of maintaining the correct signal color.

#### The Problem
- Border animation would show correct color initially
- After hovering or after some time, it would switch to green
- CSS hover styles were overriding the animation border-color

#### The Fix
- Explicit border-color set on each `.stat-card.blink-X` class
- Hover styles now explicitly maintain the signal color
- Removed problematic `!important` declarations in keyframes
- Cleaner CSS structure with proper specificity
